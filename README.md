DaCe-Fortran - Fortran Frontend for DaCe
========================================

Fortran frontend for the [DaCe](https://github.com/spcl/dace) data-centric framework. DaCe-Fortran lowers Fortran
through [flang-new](https://flang.llvm.org)'s HLFIR and an MLIR/C++ bridge into a `dace.SDFG`, so existing Fortran
kernels (ICON, CLOUDSC, ...) can be optimized and code-generated by DaCe. DaCe stays the dependency; this package is
the frontend plugin on top of it.

Quick Start
-----------

1. Clone this repository:
   ```bash
   git clone https://github.com/spcl/dace-fortran.git
   cd dace-fortran
   ```
2. Install the package (this will also install the correct version of DaCe from GitHub):
   ```bash
   pip install .
   ```
   Or for development:
   ```bash
   pip install -e ".[dev]"
   ```
3. Use it to build an SDFG:
   ```python
   import dace_fortran
   sdfg = dace_fortran.generate_sdfg("kernel.hlfir")   # -> dace.SDFG
   ```

The C++ HLFIR bridge is compiled on first use (first access of `generate_sdfg` / `SDFGBuilder`), not at import time, so
`import dace_fortran` stays cheap.

Requirements
------------

* `flang-new-21` (or `-20`) on `PATH` for `-fc1 -emit-hlfir` lowering.
* `gfortran` for the f2py/gfortran numerical references the test suite compares against.
* An LLVM/MLIR install matching the flang version (the bridge is built via CMake on first use; see
  [`dace_fortran/build_bridge.py`](dace_fortran/build_bridge.py) and [`dace_fortran/README.md`](dace_fortran/README.md)
  for the pipeline and build details).

Testing
-------

```bash
pip install -e ".[testing]"
pytest -n auto tests/
```
